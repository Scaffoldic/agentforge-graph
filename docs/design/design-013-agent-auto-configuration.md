## 1. Context

After `pip install`, a user still has to wire the CKG into their coding agent
by hand (edit a config file, write a valid MCP entry, point it at the repo),
and even then the agent under-uses the graph. feat-013 ships `ckg setup` to do
the wiring (and, with `--hooks`, the nudging) for them, plus blesses the
`uvx`/`pipx` zero-install trial.

`setup` is the deliberate **ADR-0001 framework layer** alongside `serve` — it
may import `agentforge`/framework helpers; the deterministic engine
(`core`/`ingest`/`store`/`retrieve`) stays untouched. It writes/edits files the
user did not author, so the load-bearing design property is **safety**:
structural edits, managed markers, dry-run-by-default, reversible `--undo`.

**Grounding (verified in-repo):**
- CLI is **argparse** (`src/agentforge_graph/cli.py`): subcommands are
  `sub.add_parser(name)` + `set_defaults(func=_handler)`; each handler is
  `async def _name(args) -> int`. `--version`/`-V` already exists
  (`action="version"`, `f"ckg {__version__}"`, line ~772).
- `__version__` = `importlib.metadata.version("agentforge-graph")`
  (`src/agentforge_graph/__init__.py`), `"0.0.0+unknown"` fallback.
- Config blocks are `_Block` pydantic dataclasses in `config.py` with a
  `KEY: ClassVar[str]`; `block_keys()` auto-discovers every subclass; read via
  `_read_block(Model, "key", config)` over `resolve_config()` (plain pyyaml,
  no `agentforge` import — ADR-0001). `ServeConfig(KEY="serve")` is the model
  to mirror.
- `serve-mcp` is invoked `ckg serve-mcp --repo .` → the exact `command`/`args`
  the generated entry must reproduce.
- `docs/runbooks/` is **AGENTFORGE-MANAGED** template content
  (`template:minimal@0.3.1`, regenerated by `agentforge upgrade`) — feat-013
  docs therefore live in the CKG-owned `docs/guides/`, **not** there (§4.7).

## 2. Goals

- `agentforge_graph.setup` — `ckg setup` that detects installed agents and
  writes the MCP entry, defaulting to a **repo-root `.mcp.json`** (project
  scope), `--scope user` for the global config.
- **Registry of agent adapters** (Claude Code first), mirroring the ENH-003
  provider / ADR-0006 storage registries — a new agent is one adapter + goldens.
- **Structural, marker-wrapped, reversible** config edits: dry-run + confirm by
  default, `--print`/`--yes`/`--undo`, post-write connection check.
- `--hooks` (chunk 2): append a managed nudge block to `AGENTS.md`/`CLAUDE.md`.
- FA-001 Phase 1: `--version` reports the install **channel**; bless + smoke
  `uvx`/`pipx run`; ship the consumer **guide**.
- ≥90% coverage (golden/idempotency/undo/format-preservation driven directly;
  the live connection check + trial-path smoke are env-gated/CI); `mypy
  --strict`; ruff.

## 3. Non-goals

- FA-001 Phases 2–3 (Homebrew, `install.sh`/`.ps1`, self-contained bundle,
  `ckg self-update`); FA-004 local embeddings. Installing the agents
  themselves. **Blocking** policy hooks. A GUI. Multi-format *user-scope*
  adapters beyond Claude Code's JSON (TOML/YAML adapters land with the agents
  that need them).

## 4. Proposal

### 4.1 Package layout

```
src/agentforge_graph/setup/
  __init__.py     # public: run_setup(), SetupPlan, register_adapter, builtins
  plan.py         # SetupPlan / SetupAction (the diff model) + render
  registry.py     # AgentAdapter protocol + registry (register/get/all)
  merge.py        # JsonMcpMerger: structural .mcp.json/~.claude.json edits + markers + undo
  hooks.py        # (chunk 2) AGENTS.md/CLAUDE.md managed-block append/undo
  check.py        # post-write MCP stdio connection check (spawn → initialize → ckg_status)
  channel.py      # FA-001 P1: best-effort install-channel detection for --version
  adapters/
    __init__.py
    claude_code.py  # detection + user-scope ~/.claude.json target
    mcp_json.py     # generic project-scope .mcp.json target (the standard)
src/agentforge_graph/
  config.py       # + SetupConfig(_Block)
  cli.py          # + `ckg setup`; extend the --version string with the channel
```

`setup` **imports `agentforge`/`mcp`** (framework layer, ADR-0001 exception) —
no layering test applies (engine packages keep theirs). The `mcp` SDK is
already a base dep (agentforge-mcp), so `check.py`'s client needs no new dep.

### 4.2 Config (`SetupConfig`)

A new `_Block` (auto-registered via `block_keys()`), read with
`_read_block(SetupConfig, "setup", cfg)`:

```python
class SetupConfig(_Block):
    """The ``setup:`` block (feat-013 — agent auto-configuration)."""
    KEY: ClassVar[str] = "setup"
    scope: str = "project"      # project (repo .mcp.json) | user (~/.claude.json)
    transport: str = "stdio"    # stdio | http (mirrors ServeConfig)
    install_hooks: bool = False # opt-in AGENTS.md/CLAUDE.md nudge block
    agents: list[str] = Field(default_factory=list)  # [] = auto-detect all; else allowlist
```

CLI flags override config (`--scope`, `--transport`, `--hooks`, `--agent`).

### 4.3 Adapter registry + detection (`registry.py`, `adapters/`)

Mirrors the provider/storage registries — core flow is agent-agnostic; each
agent is data + a small adapter:

```python
@dataclass(frozen=True)
class AgentTarget:
    key: str            # "claude-code"
    display: str        # "Claude Code"

class Detection(TypedDict):
    installed: bool
    note: str           # where it was found, or why skipped

class AgentAdapter(Protocol):
    target: AgentTarget
    def detect(self) -> Detection: ...
    # File this agent reads for the requested scope (None ⇒ scope unsupported).
    def config_path(self, repo: Path, scope: str) -> Path | None: ...

def register_adapter(a: AgentAdapter) -> None: ...
def adapters(allowlist: list[str]) -> list[AgentAdapter]: ...   # [] ⇒ all builtins
```

- **`mcp_json` (project scope, default):** the target file is `<repo>/.mcp.json`
  — the portable MCP standard. Detection here is "does any known MCP client
  exist" (informational); the file is written regardless because committing it
  wires every `.mcp.json`-aware agent.
- **`claude_code` (user scope):** `detect()` probes `~/.claude.json` or the
  `claude` binary on PATH; `config_path(scope="user")` → `~/.claude.json`;
  `config_path(scope="project")` → `<repo>/.mcp.json` (Claude Code reads it).
- Conservative detection: ambiguous ⇒ `installed: False` + a "skipped" note,
  **never** a write. Unknown agents are listed, never guessed.

v1 ships `mcp_json` + `claude_code`. A second agent = one `adapters/<x>.py` +
goldens.

### 4.4 Structural MCP merge + markers + undo (`merge.py`)

Both `.mcp.json` and `~/.claude.json` are JSON holding an `mcpServers` map, so
one merger serves both:

```python
ENTRY = {
  "command": "ckg",
  "args": ["serve-mcp", "--repo", "<rel-or-given>"],   # stdio default
  "_managed_by": "agentforge-graph",                    # the marker
}
```

- **Read → parse → merge our `mcpServers.ckg` key → serialize**, never string
  splice. Preserve unknown keys, key order, and (for JSONC, if a target uses
  it) comments via a comment-preserving parse; v1 targets are plain JSON.
- **Idempotent:** writing twice yields a byte-identical file (we set, not
  append). Re-running reconciles the entry to desired state.
- **`--undo`:** remove `mcpServers.ckg` **only if** it carries
  `_managed_by: agentforge-graph` (never touch a user's own `ckg` entry). If we
  created the file and it's left with no servers, remove the file; otherwise
  leave the rest untouched.
- **Transport:** stdio entry by default; `--transport http` emits a `url`
  entry and **refuses a non-loopback host without a token** (carries ENH-005's
  bind-safety into generated config).
- **Path:** project scope writes `--repo .` (relative → portable in a committed
  file); user scope writes the absolute repo path (a global file serves a
  specific repo).

### 4.5 `--version` channel stamp (`channel.py`, FA-001 P1)

Best-effort, **diagnostic only** (never changes behavior): inspect
`sys.prefix`/`sys.executable` for markers — a uv tool cache (`/uv/`,
`archive-v0`) ⇒ `uvx`; a pipx venv (`/pipx/`) ⇒ `pipx`; else `pip`. Extend the
existing version action to `f"ckg {__version__} ({channel})"`. Unknown ⇒ omit
the suffix rather than guess. Small, isolated, and unit-tested with faked
paths.

### 4.6 CLI (`ckg setup`)

```
ckg setup [path] [--scope project|user] [--agent NAME] [--transport stdio|http]
          [--hooks] [--print] [--yes] [--no-check] [--undo]
```

`_setup(args)` flow:
1. Resolve repo (reuse `_resolve_repo_path` / ENH-019 discovery) + load
   `SetupConfig` (flags override).
2. Build the `SetupPlan`: for each selected adapter, compute its
   `config_path(repo, scope)` + the diff (add/replace `ckg` entry; with
   `--hooks`, the AGENTS.md block).
3. **Render the plan/diff** (detected agents, target files, `+`/`~` lines).
4. `--print` ⇒ stop here (write nothing). Else **prompt `Apply? [y/N]`**;
   `--yes` skips the prompt. **Stateless — no remembered consent.**
5. Apply via `JsonMcpMerger` (+ `hooks` if `--hooks`).
6. Unless `--no-check`, run the **connection check** (§4.8) and report
   `connected ✓` / a clear failure.
7. `--undo` is its own path: compute what we'd remove (marked entries/blocks)
   and reverse them.

Returns `0` on success/clean dry-run, non-zero on a write/merge/check failure
(with an actionable message, never a stack trace).

### 4.7 Nudge hooks (`hooks.py`, chunk 2)

Append a **managed block** to the repo's `AGENTS.md` (and/or `CLAUDE.md` if
present) — repo-level, portable, non-blocking:

```
<!-- agentforge-graph:start -->
## Prefer the code graph
For structural questions (callers, impact, routes, where-defined), use the
ckg_* MCP tools instead of grep/glob — cheaper and grounded. Fall back to file
reads only when the graph can't answer.
<!-- agentforge-graph:end -->
```

Idempotent (replace the marked block in place) and `--undo`-removable (delete
exactly the marked block). No file ⇒ create a minimal `AGENTS.md` with just the
block. An active Claude-Code `PreToolUse` hook was rejected for v1 (Claude-only,
noisy per-search).

**Docs (this feat's definition of done, §12 of the spec):** a CKG-owned guide
`docs/guides/11-agent-auto-configuration.md` carrying both the **how-to**
(trial → setup → scopes → hooks → undo, with troubleshooting) **and** the
operational **reference** content (exactly what gets written per scope, the
marker format, undo/upgrade behavior, safety guarantees). It is *not* placed
under the framework-managed `docs/runbooks/`. README quickstart + guide-10
cross-link to it.

### 4.8 Connection check (`check.py`)

Spawn the configured command (`ckg serve-mcp --repo <repo>`) as an MCP **stdio
subprocess**, run an MCP `initialize` + `tools/list` via the `mcp` SDK client,
assert `ckg_status` is present (optionally call it), then tear down. Bounded by
a wall-clock timeout; any failure → a clear "wrote config, but the server
didn't answer — try `ckg serve-mcp --repo .` manually" message. Env-gated in
CI (spawns a real subprocess), skippable with `--no-check`.

## 5. Alternatives considered

| Option | Why not |
|---|---|
| Shell out to `claude mcp add …` | Requires `claude` on PATH, Claude-only, and cedes marker/undo control. We own a structural JSON merge that generalizes to any `.mcp.json` client. |
| Global `~/.claude.json` as the default | The graph is per-repo; a global entry can only serve one absolute path and isn't shareable. Project `.mcp.json` matches the engine 1:1 and is committable. Global stays available via `--scope user`. |
| Remember consent after first apply | Adds persisted state + a corruption/edge surface for one saved keystroke. Stateless confirm (always ask unless `--yes`) is simpler and safer. |
| Active `PreToolUse` nudge hook | Claude-specific and noisy on every grep. A managed `AGENTS.md` block is portable and inherently non-blocking. |
| Put `setup` in the deterministic engine | It imports framework/MCP; it is the ADR-0001 framework layer like `serve`. Engine stays clean. |
| Text-splice the config | One malformed splice corrupts a user's file. Structural parse/merge + markers is the whole safety story. |

## 6. Migration / rollout

Additive: a new subcommand + a new `setup:` config block + a new
framework-layer package. No engine change, no persisted-graph change, no schema
bump. `setup.scope`/flags are forward-compatible (new agents = new adapters =
minor). The generated `.mcp.json` is a new repo file the user chooses to commit.

## 7. Risks

| Risk | Mitigation |
|---|---|
| Corrupting a user's hand-edited config | Structural parse/merge only; managed marker; dry-run + confirm default; `--undo`; golden + idempotency + format-preservation tests. |
| Over-broad detection (claims an agent that isn't there) | Conservative probes; ambiguous ⇒ skipped + reported, never written. |
| `.mcp.json` already has a user `ckg` entry | Only replace/remove entries carrying our `_managed_by` marker; a user's own `ckg` is left intact (and reported). |
| `--transport http` foot-gun | Refuse a non-loopback host with no token (ENH-005 guard) at plan time. |
| Channel detection wrong | Diagnostic-only, never gates behavior; omit the suffix when unsure. |
| Connection check flaky in CI | Env-gated; `--no-check` escape; bounded timeout; failure is a warning with a manual fallback, not a hard setup failure. |
| `agentforge`/`mcp` untyped under mypy --strict | Already in `ignore_missing_imports` from `serve`; extend if needed. |
